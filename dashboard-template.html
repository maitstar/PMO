      📋 Data Dictionary & Methodology
      How each metric is calculated and where the data comes from
      1️⃣ Completion Progress (%)
      Three lines: Planned vs. Predicted vs. Actual
      Line Source Variable Calculation
      Planned project.startDate, project.targetDate Linear interpolation from project start to target date. Assumes 0% on start date → 100% on target date.
      Actual issue.completedAt (timestamp) Count issues with completedAt in each month / total issues created up to that month. Only counted for months up to today.
      Predicted issue.completedAt (last 30 days) Velocity = (issues completed in last 30 days) / 30 = issues/day. Future months = (current completed) + (velocity × days ahead), capped at 100%.
      // From fetch-linear.mjs, deriveMetrics() const velocity = recentCompleted / 30; // issues/day const projected = completedSoFar + (velocity * daysAhead); predicted = Math.min(100, Math.round((projected / totalIssues) * 100));
      2️⃣ Resource Allocation (%)
      Which projects consume effort right now
      Metric Source Variable Calculation
      Active Issues per Project issue.state.type === 'started' + issue.project.name Filter all issues where state.type = 'started' (in progress). Group by project. Count per project.
      Percentage Count per project Each project's count / total active issues across all projects.
      const activeIssues = issues.filter(i => i.state.type === 'started'); const projectCounts = {}; activeIssues.forEach(iss => { const key = iss.project?.name || 'Unassigned'; projectCounts[key]++; }); const totalActive = activeIssues.length; const resourceAllocation = Object.entries(projectCounts) .map(([name, count]) => ({ name, count, pct: (count / totalActive) * 100 }));
      3️⃣ Risk Distribution
      High / Medium / Low risk issues
      Risk Level Filter Criteria Source Variables
      High Urgent priority issues OR (High priority + overdue) issue.priority <= 1 OR (priority === 2 AND dueDate < today)
      Medium (High priority + no assignee) OR (Medium priority + overdue) (priority === 2 AND !assignee) OR (priority === 3 AND dueDate < today)
      Low Upcoming issues (due within 14 days, not started) state.type === 'backlog' AND dueDate is 0-14 days away
      // Priority mapping: 0=None, 1=Urgent, 2=High, 3=Medium, 4=Low const highRisk = openIssues.filter(iss => iss.priority <= 1 || (iss.priority === 2 && iss.dueDate && new Date(iss.dueDate) < today) ); const mediumRisk = openIssues.filter(iss => (iss.priority === 2 && !iss.assignee) || (iss.priority === 3 && iss.dueDate && new Date(iss.dueDate) < today) );
      4️⃣ Load Balance
      Work per team member
      Column Source Variable Calculation
      Total Open issue.assignee.name + state.type !== 'completed' Count all non-completed issues assigned to each person.
      In Progress issue.state.type === 'started' Count only 'started' (in-progress) issues per person.
      P1/P2 Open issue.priority <= 2 Count open (not completed) issues with priority = Urgent or High.
      Workload % Count in-progress per person Each person's active count / max(active counts) × 100. Shows relative workload.
      5️⃣ Projects Summary Table
      All projects with status and deadline
      Column Source Variable Notes
      Progress % project.progress (Linear native) Linear calculates this based on issue completion within the project.
      Open Issues issue.project.id + state.type !== 'completed' Count issues in this project that are not completed.
      P1/P2 Open issue.priority <= 2 + same project Count critical/high priority blockers in this project.
      Days Left project.targetDate Ceil((targetDate - today) / 86400000). Negative = overdue.
      📡 Data Source: Linear API
      All data comes from Linear's GraphQL API
      // fetch-linear.mjs queries: ISSUES_QUERY: issues(filter: { state: { type: { nin: ["cancelled"] } } }) PROJECTS_QUERY: projects(first: 50) TEAMS_QUERY: teams(first: 20)

      Key fields pulled: issue.id, title, priority, state.type, assignee, dueDate, completedAt, createdAt, project.id, project.name, project.progress, project.targetDate, project.startDate
